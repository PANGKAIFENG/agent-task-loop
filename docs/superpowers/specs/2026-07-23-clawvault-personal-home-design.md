# ClawVault 个人首页设计规格

## 目标

将 ATL 的只读「个人工作贡献」视图升级为 ClawVault 的个人首页，首屏先回答：

1. 最近一段时间是否持续推进；
2. 今天和本周完成了什么；
3. 当前还有哪些任务需要人工判断或继续推进；
4. Agent 与 Token 的投入是否有可核对记录。

## 信息架构

首页保留 Obsidian 的原生工作区入口，视图内部提供四个轻量视图：

- `总览`：Personal Pulse、当前推进 Top 3、任务流状态、下一项行动和输入队列；
- `今日`：当前日期可推进的任务列表；
- `输入`：ATL Inbox 中等待人工判断的任务；
- `复盘`：选中日期的项目、任务产出和趋势明细。

## 数据边界

- 任务完成只来自 ATL Audit 中可核对的完成事件；不使用 `updated_at` 推断完成日期。
- 有效产出只统计完成任务的 Artifact 引用。
- Agent 投入使用 OpenToken 的每日 Normalized Token 快照；Token 不按项目归因。
- 文章消费状态当前没有稳定的持久化字段，Personal Pulse 中保留「消费」入口但显示“待接入”，不填充推测数字。
- Top 3 先展示真实的 `in_progress` 与 `ready` 任务作为当前推进候选，不把它们冒充用户手工目标。
- Inbox、任务状态和项目名称全部从当前 Vault 读取；入口和任务卡片保持只读，打开任务仍跳转到原 Markdown 文件。

## Personal Pulse

- 默认范围为最近 `26 周`，保留 `7 天 / 26 周 / 1 年`切换；默认展示在总览第一块。
- 模式为 `任务 / 消费 / 产出 / AI`：任务使用完成任务密度，产出使用带 Artifact 的完成任务，AI 使用每日 Normalized Token，消费显示待接入状态。
- 右侧摘要展示连续推进、本周完成、今日完成和历史完成日期缺口。
- 下方三条趋势展示完成任务、有效产出和 Normalized Token 的总量及折线。

## 非目标

- 本次不实现文章消费判定、不修改 TaskNotes 内部状态、不新增 Agent 自动执行权限、不修改用户任务内容。
- 本次不把视图强制写入 Obsidian workspace 配置；通过现有 Ribbon 图标和命令面板打开，避免覆盖用户布局。

## 验收标准

1. 真实 Obsidian 中入口名称为「ATL：个人首页」，打开后 Personal Pulse 位于总览首屏顶部。
2. 默认范围为 26 周，热力图按周列正确排列，点击日期仍能查看真实项目与产出。
3. 任务完成、有效产出、Token、Inbox 和状态统计均来自真实数据；OpenToken 缺失时任务区仍可用。
4. 文章消费入口明确显示待接入，不出现虚构的消费数量。
5. 桌面和窄窗口不发生横向布局破坏；原有视图、任务跳转、刷新和设置入口测试保持通过。
