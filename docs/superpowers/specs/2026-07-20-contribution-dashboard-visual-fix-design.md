# 个人工作贡献首页视觉修复设计

## 目标

修复 ATL Obsidian“个人工作贡献”首页中贡献热力图纵向堆叠的问题，并把页头副文案改得更贴近用户真正要回看的信息。

## 已确认方案

- 副文案改为：`看见每天完成了什么，也看见时间花在了哪里。`
- 热力图继续使用 GitHub 式周历结构：横向为周，纵向为周一到周日。
- `7 天`、`12 周`、`1 年`只改变数据范围，不改变网格规则。
- 日期按钮继续保留原有点击、键盘焦点、Tooltip、选中态与完成强度。

## 根因与实现

当前日期按钮只设置了 `grid-row`。因为不同星期可以同时占据第一列，CSS Grid 自动放置不会主动把后续日期推进到下一周，最终形成单列。

渲染时以返回日期数组的第一天为锚点，同时计算每个日期的星期行和周列：

```text
row = Monday-based weekday(date)
column = floor((firstDayRow - 1 + dayIndex) / 7) + 1
```

DOM 明确写入 `grid-row` 和 `grid-column`，CSS 继续只负责 14px 单元格、5px 间距和横向滚动。

## 范围边界

本次不修改贡献数据口径、日期范围长度、KPI、趋势图、OpenToken、任务状态或 Vault 内容，也不做额外视觉重构。

## 验收

- 同一周内不同星期处在同一列。
- 跨过周日后，星期一进入下一列。
- 不同起始星期的日期范围均正确布局。
- 新副文案显示正确。
- 原有交互与测试保持通过。
- 在真实 Obsidian 中验证三个范围，验收过程只读任务数据。
