# Calendar Event Layout Design

## Problem

TaskNotes uses FullCalendar's overlapping time-grid layout by default. When two or more tasks or imported DingTalk events share a time range, later cards cover part of earlier cards. TaskNotes also allows calendar titles to wrap with visible overflow, so long Chinese titles can leave a narrow event card and collide with adjacent content.

The problem affects the calendar view embedded in `10_Tasks/Views/任务总看板.base` and the ATL-generated `10_Tasks/Views/统一日历.base`.

## Goals

- Display concurrent time-grid events in separate horizontal lanes without visual overlap.
- Keep every event title inside its card and show an ellipsis when it does not fit.
- Preserve event time, click, drag, and resize behavior.
- Apply the same behavior to local tasks and read-only DingTalk copies.
- Keep TaskNotes as an independent dependency; ATL must not fork or modify TaskNotes plugin files.

## Non-Goals

- Replacing TaskNotes or FullCalendar.
- Changing task dates, event durations, synchronization behavior, or DingTalk data.
- Adding a custom calendar renderer or DOM mutation observer.
- Solving dense calendars by hiding events behind a new aggregation interaction.

## Design

### Concurrent event layout

ATL will set `options.slotEventOverlap: false` on both calendar definitions that it manages:

1. The calendar view generated by `UnifiedCalendarController`.
2. The calendar view updated by `BoardAppearanceController.applyRecommendedPreset`.

TaskNotes exposes this option through its Base calendar schema and passes it to FullCalendar. FullCalendar then calculates non-overlapping lane widths while retaining its native event ordering, dragging, resizing, and hit targets.

`BoardAppearanceController` will update exactly one `tasknotesCalendar` view in the task board. It will accept the existing names `日历视图` and `日历`, reject an ambiguous board containing multiple matching calendar views, and preserve the existing byte-for-byte backup behavior before writing. Preset status will include the calendar option so an older board is reported as needing the updated preset.

### Title containment

ATL's existing `atl-task-card-theme` body class will also scope calendar styling. For TaskNotes time-grid events, ATL will constrain the event container and its FullCalendar content frames with `min-width: 0` and `overflow: hidden`. The title will use one line with `white-space: nowrap`, `text-overflow: ellipsis`, and hidden overflow.

The event time remains visible as a separate line when TaskNotes renders it. Users can open the event to read the complete title; ATL will not mutate TaskNotes DOM or inject a second tooltip system.

### Existing and new views

New ATL unified calendars receive the option at creation time. Existing task-board calendars receive it when the user applies the ATL recommended layout, using the current backup and restore workflow. During this release's local verification, the updated preset will be applied to the test Vault through the Obsidian UI.

## Error Handling

- A task board with no unique managed Kanban view continues to fail without writing.
- A task board with more than one supported TaskNotes calendar view fails as ambiguous without writing.
- A task board with no calendar view keeps its Kanban preset behavior; ATL does not create an unexpected view in that file.
- Existing symlink and Vault-boundary protections remain unchanged.

## Verification

Automated tests will prove:

- A newly generated unified calendar sets `slotEventOverlap: false`.
- Applying the recommended board preset sets the same option on the existing calendar view.
- Preset status distinguishes an older overlapping calendar from the updated layout.
- Ambiguous calendar views are rejected without modifying the Base or backup.
- The built plugin stylesheet contains the scoped containment and ellipsis rules.

Manual Obsidian verification will cover:

- A long event title remains inside its card with an ellipsis.
- Two or more concurrent events display side by side without covering each other.
- Clicking, dragging, and resizing events still work.
- Local ATL tasks and imported DingTalk events appear in the same calendar with the same layout rules.
