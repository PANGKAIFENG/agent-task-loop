# 在 Obsidian 中确认 ATL 任务

ATL Obsidian 插件与 TaskNotes 配合使用：TaskNotes 负责看板和任务卡片，ATL 插件负责补齐任务信息并安全地移动任务。它不是 TaskNotes 的 fork，也不会修改 TaskNotes。

## 安装

MVP 阶段尚未进入 Obsidian 社区插件市场，需要通过 Finder 安装一次：

1. 获取 ATL 插件发布包，确认其中包含 `main.js`、`manifest.json`、`styles.css`。
2. 在 Finder 中打开你的 Vault，显示隐藏文件后进入 `.obsidian/plugins/`。
3. 新建 `agent-task-loop` 文件夹，把三个文件放进去。
4. 重启 Obsidian。
5. 打开“设置 → 第三方插件”，启用 `Agent Task Loop`。
6. 打开“设置 → Agent Task Loop”，开启“允许 ATL 管理此 Vault”。

以上过程不需要终端。授权默认关闭，开启后只允许 ATL 管理当前 Vault 的 `10_Tasks` 目录。

## 日常确认任务

1. 打开 TaskNotes 的任务总看板。
2. 在“收件箱”中找到需要处理的任务卡片。
3. 打开卡片的文件菜单，选择“确认并移到待执行”。
4. 在居中弹窗中选择项目，或新建项目。
5. 填写任务目标和至少一条验收标准。
6. 设置优先级。
7. 决定是否开启“允许 Agent 自动执行”。
8. 点击“确认并移到待执行”。

确认成功后，任务进入“待执行”。如果关闭 Agent 自动执行，任务仍会进入待执行，但自动调度器不会领取它。

也可以先打开任务文件，再从 Obsidian 命令面板执行“Agent Task Loop: 确认当前任务并移到待执行”。

## 项目怎么选

- 已有项目：下拉列表会显示 `10_Tasks/Projects` 中的项目，直接选择即可。
- 新项目：选择“新建项目...”，填写名称和说明。插件会创建项目文件，再关联当前任务。
- 项目用于聚合同一目标下的任务和后续调研结果，不需要为每个任务都新建项目。

## 字段怎么写

- 任务目标：写清楚最终要回答的问题，例如“梳理产品定位、核心能力和公开定价”。
- 验收标准：写成可以明确检查的结果，例如“至少引用一个官方页面”“明确区分事实和待验证信息”。
- 优先级：只影响队列排序，不会放宽 Agent 权限。
- 允许 Agent 自动执行：只读调研可开启；仍需人工判断或涉及外部操作的任务应关闭。

## 常见提示

- “请先允许 ATL 管理此 Vault”：到插件设置中开启授权。
- “任务已经不在收件箱”：TaskNotes 画面可能尚未刷新，重新打开看板。
- “同名项目已经存在”：关闭弹窗后重新打开，并从已有项目中选择。
- “任务刚刚被其他操作修改”：刷新任务后再确认，ATL 不会覆盖并行修改。

## 当前边界

这个插件版本解决的是 Obsidian 内的任务确认和项目创建。TaskNotes 的看板、筛选、日历仍由 TaskNotes 提供。Agent 运行环境和自动调度器的首次部署仍属于高级维护能力，尚未包含在这个插件安装包中。
