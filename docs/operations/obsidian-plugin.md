# Obsidian 用户操作指南

Agent Task Loop（ATL）负责收集、去重、任务文件和可选的 AI 能力；TaskNotes 负责看板、卡片、筛选和日历。两者通过 Markdown 属性协作，ATL 不是 TaskNotes 的 fork。

## 先理解两种用法

### 普通手工待办

```text
收件箱 → 待办 → 进行中 → 已完成
```

你自己决定任务是否要做、什么时候做和什么时候完成。普通任务只需要标题，不要求项目、目标、完成条件或执行权限。

### 自动调研任务

```text
收件箱 → 待办 → Agent 执行中 → 待验收
```

只有你明确补齐项目、目标、完成条件和只读权限，并开启自动执行后，ATL Runner 才能领取。自动化是增强能力，不是使用看板的前提。

## 安装插件

1. 从项目 [Releases](https://github.com/PANGKAIFENG/agent-task-loop/releases) 下载最新插件压缩包。
2. 解压后确认有 `main.js`、`manifest.json`、`styles.css`、`atl-runner.mjs`。
3. 在 Finder 中打开 Vault，按 `Command + Shift + .` 显示隐藏文件。
4. 进入 `.obsidian/plugins/agent-task-loop/`；没有该文件夹时直接新建。
5. 放入四个文件并重启 Obsidian。
6. 打开“设置 → 第三方插件”，启用 `Agent Task Loop`。

安装和日常手工管理不需要终端。推荐同时安装 TaskNotes。

## 第一次配置

### 1. 开启 Vault 管理

打开“设置 → Agent Task Loop”，开启“允许 ATL 管理此 Vault”。ATL 只管理 `10_Tasks` 中的任务数据和自己的后台配置，不会修改其他 Obsidian 笔记。

### 2. 应用人工任务看板

在“任务看板”中点击“应用人工任务布局”。ATL 会把 `10_Tasks/Views/任务总看板.base` 设置为按原始 `status` 分组的四列看板：

| 显示名称 | status 值 | 用途 |
| --- | --- | --- |
| 收件箱 | `inbox` | 新收集、尚未决定 |
| 待办 | `ready` | 决定要做，尚未开始 |
| 进行中 | `in_progress` | 当前正在处理 |
| 已完成 | `done` | 已经完成 |

首次应用会生成 `.atl-backup`，点击“恢复原布局”可恢复原 Base。

### 3. 自定义状态

四列只是默认值。需要“等待回复”“以后再做”等列时，在 TaskNotes 的状态设置中增加状态值和显示名称，再把它加入相应视图。ATL 支持安全的自定义状态字符串并原样保存。

注意：Inbox 任务第一次进入日常管理时，建议使用 ATL 的“移到待办”，让任务文件和确认记录一起更新。进入待办后，可在 TaskNotes 中人工拖动状态。

## 收集任务

### 随手新建

1. 点击 Obsidian 左侧“ATL：新建任务”。
2. 填写标题；说明和优先级可选。
3. 点击“加入 Inbox”。

### 从笔记同步助手实时获取

1. 把想法或待办发给“笔记同步助手”，等待它同步到当前 Vault。
2. 点击左侧“ATL：从同步助手获取待办”。
3. ATL 只读扫描新记录，并调用已配置的模型提取候选。
4. 在候选弹窗中勾选真正要进入 Inbox 的项目。
5. 点击“将所选任务加入 Inbox”。

首次扫描覆盖昨天和今天；之后从上次成功扫描所在日期补扫到今天。只有提交成功才推进扫描进度，取消、模型失败或写入失败不会跳过内容。

### 与每日复盘如何去重

实时扫描和每日复盘都使用 ATL 收集服务：

1. 优先匹配稳定 `sourceKey`；
2. 再比较来源笔记、任务标题和原文证据；
3. 确认是同一行动时返回已有任务；
4. 仅标题相似但证据不足时保留两条，并标记疑似重复。

因此，无论每日复盘还是实时扫描先运行，同一来源的同一行动只保留一条任务。

## 把 Inbox 任务移到待办

1. 打开 TaskNotes 的任务总看板。
2. 在“收件箱”中找到任务并打开对应 Markdown 文件。
3. 点击文件右上角菜单，或在文件列表中右键任务文件。
4. 选择“移到待办”。
5. 需要时选择项目、补充目标和完成条件；这些字段都可以留空。
6. 点击“移到待办”。

如果任务文件已打开，也可从命令面板运行“Agent Task Loop: 将当前任务移到待办”。

### 项目什么时候填写

项目用于聚合同一长期目标下的任务和共享上下文，不是必填分类标签。

- 临时个人待办：选择“暂不选择项目”。
- 已属于持续项目：选择已有项目。
- 确实出现新的长期目标：选择“新建项目”。

## 使用 AI 帮我整理

当标题太短、还说不清目标时：

1. 在“移到待办”弹窗的“补充说明”中写一句你最终想得到什么。
2. 点击“AI 帮我整理”。
3. 等待目标和 1 至 5 条完成条件自动回填。
4. 人工修改结果，再点击“移到待办”。

AI 只整理当前任务信息，不执行任务、不读其他文件，也不替你选择项目和优先级。整理失败时仍可直接移到待办。

模型沿用“设置 → Agent Task Loop → 模型服务”的配置：

| 模式 | 适合场景 |
| --- | --- |
| 沿用 Claude Code 当前配置 | 已用 Claude Code 或 CC-Switch 管理模型，推荐 |
| 自定义服务 | 只让 ATL 使用指定 Model 和 Base URL |

ATL 不保存 API Key。登录和密钥由 Claude Code 或系统环境管理。

## 人工推进任务

任务进入待办后，可以直接使用 TaskNotes：

1. 决定开始时，把卡片拖到“进行中”。
2. 在 TaskNotes 日历中设置 `scheduled` 日期和时间，安排今天什么时候做。
3. 暂时无法继续时，可放到自己新增的“等待回复”等状态。
4. 完成后，把卡片拖到“已完成”。

ATL 会保留 TaskNotes 写入的 `scheduled`、`due` 和其他未知 frontmatter。

## 同步钉钉日历到 TaskNotes

这项能力适合把公司日程放进同一个时间规划视图，同时保持钉钉为唯一远端事实源。

### 第一次连接

1. 打开“设置 → Agent Task Loop → 钉钉日历”。
2. 填写钉钉 CalDAV 地址、账号和密码。
3. 点击“测试连接”，确认插件识别到主日历。
4. 开启“启用日历同步”。
5. 点击“立即同步”。
6. 打开 TaskNotes 日历，查看从今天到未来 90 天的钉钉日程。

密码保存到 Obsidian `SecretStorage`，不会出现在 `data.json`。已有的 macOS Keychain 凭据会在首次读取时安全迁移；安装和日常使用都不需要终端。

### 在 TaskNotes 中怎么安排

每个日程会成为 `TaskNotes/DingTalk/` 下的独立 Markdown 文件。它使用：

- `scheduled`：日程日期和开始时间；
- `timeEstimate`：日程持续分钟数；
- `origin: dingtalk_caldav`：标识钉钉来源；
- `dingtalk_event_key_hash`：稳定去重标识。

你可以在 TaskNotes 日历中拖动本地副本，也可以补项目、标签、优先级、状态和本地备注。它不会显示在 ATL 的 `10_Tasks` 收件箱、待办看板或 Agent 执行队列中。

### 同步冲突怎么处理

| 情况 | 下次同步结果 |
| --- | --- |
| 你在 TaskNotes 拖动了时间，钉钉时间没变 | 保留本地时间 |
| 钉钉修改了时间或时长 | 更新本地 `scheduled` / `timeEstimate`，钉钉优先 |
| 你补了项目、标签、优先级、状态或备注 | 始终保留 |
| 钉钉取消日程 | 本地副本标记为 `cancelled`，不删除文件 |
| 你删除了本地副本 | 记录 tombstone，不会自动重新导入 |
| 周期日程 | 每次 occurrence 是独立的本地文件，可以分别拖动 |

无论发生哪种情况，ATL 都不会向钉钉发送 `PUT`、`POST`、`PATCH` 或 `DELETE`，也不会创建钉钉待办。

### 手动同步与恢复

- 手动同步：设置页点击“立即同步”，或命令面板运行“Agent Task Loop: 立即同步钉钉日历”。
- 自动同步：启用后，Obsidian 启动时一次，此后每 15 分钟一次。
- 重置 ledger：连续两次点击“清除导入记录”。已有 `TaskNotes/DingTalk/` 文件不会被删除。
- 连接失败：检查 HTTPS CalDAV 地址、账号、密码和网络后重新“测试连接”。错误提示不会显示密码或日历内容。

清除导入记录会解除 tombstone 和旧远端快照。它用于明确的恢复场景，不是日常操作。

## 查看个人工作贡献

在 Obsidian 左侧点击图表入口“ATL：个人工作贡献”，也可以从命令面板运行“打开个人工作贡献”。这是只读的统计首页，适合每天开始或结束工作时快速回顾：

| 区域 | 含义 |
| --- | --- |
| 今日完成 | 今天有完成审计证据的不同任务数；同一任务同一天只计一次 |
| 本周完成 | 本周一至今天的完成任务数 |
| 连续完成 | 从今天（今天没有完成时从昨天）向前连续有完成记录的天数 |
| 今日 Normalized Token | 本机 OpenToken 的当天标准化 Token 汇总，不按项目归因 |
| 工作贡献 | 7 天、12 周、1 年的每日完成任务热力图；点击日期查看当天内容 |
| 趋势 | 每日完成任务和每日 Normalized Token 的折线趋势 |

任务贡献只认两类可审计事件：任务确认通过，或任务生命周期进入 `done`。不会根据任务的 `updated_at` 推断完成日。已经是 `done` 但没有对应 Audit 的历史任务会显示为“缺少可核对日期”，不计入热力图。

OpenToken 读取失败不会影响任务统计。首页会保留任务完成、项目和产出，并在 Token 区显示未检测到、缓存过期或暂时不可用。点击“查看数据源设置”可看到数据来源说明；OpenToken 需要通过你原来的安装来源（例如已有的本机插件/工具安装方式）安装或更新，ATL 不保存原始会话、API Key 或其他凭据。

首页不会自动执行任务、修改任务状态或写入 Audit。要打开某个任务或 Agent 产出，直接点击“可核对产出”中的任务标题或文件图标。

## 复制给 Codex 自己执行

1. 在任意 ATL 任务文件上打开文件菜单。
2. 选择“复制给 Codex”。
3. 打开 Codex 并粘贴。

复制内容包括绝对任务路径、正文、项目、目标、完成条件、状态和来源摘要。该动作不会启动 Codex，也不会自动修改任务；完成工作后仍由你更新任务状态。

## 启用自动调研（可选）

后台调研需要：

- Node.js 24+；
- 已安装并登录 Claude Code；
- 有效的 Model 和 Base URL 配置；
- 用户明确授权的本地资料目录。

设置页面会检查 ATL Runner、Node.js、Claude Code 和后台任务。状态为“待安装”时点击“启用后台执行”；状态为“已就绪”后，可等待整点运行或点击“立即试跑”。

Runner 只领取满足全部条件的任务：`ready`、`research`、已关联项目、有目标、有完成条件、`read_only_research`、`auto_executable=true`。普通待办不会因为进入 `ready` 就被自动领取。

成功完成后，任务进入 Review，并生成：

```text
10_Tasks/Artifacts/<任务 ID>/attempt-xxx.md
```

## 当前验收限制

底层任务服务支持验收通过、带反馈退回、标记阻塞和取消，但当前 Obsidian 插件尚未提供完整的 Review 操作按钮。现在可以查看待验收任务和 Artifact；不要手改自动调研任务的状态来代替带审计的验收流程。

| 人工决定 | 目标状态 | 当前 Obsidian 按钮 |
| --- | --- | --- |
| 验收通过 | Done | 尚未上线 |
| 要求修改 | Ready | 尚未上线 |
| 标记阻塞 | Blocked | 尚未上线 |
| 取消任务 | Cancelled | 尚未上线 |

这项限制只影响 ATL Runner 产生的待验收结果，不影响普通任务在 TaskNotes 看板中的人工管理。

## 常见提示

- **请先允许 ATL 管理此 Vault**：到插件设置开启 Vault 权限。
- **任务已经不在收件箱**：TaskNotes 可能尚未刷新，重新打开看板。
- **没有发现新的待办候选**：模型检查过新内容，但没有合格行动。
- **同步助手扫描失败**：检查 Claude Code、Model 和 Base URL 后重试；扫描进度不会被跳过。
- **AI 整理失败**：可直接人工填写或留空，不影响移到待办。
- **复制失败**：确认是桌面版 Obsidian，并重新打开任务文件。
- **Node.js 或 Claude Code 未找到**：只影响 AI 和自动调研，不影响手工任务管理。
- **后台配置冲突**：ATL 不会覆盖不受它管理的同名 macOS 服务，需要开发者检查。
- **钉钉日历连接失败**：确认使用 HTTPS CalDAV 地址，并重新输入账号密码；不会影响现有本地日历文件。
- **本地拖动后又被改回**：说明钉钉端也修改了同一日程时间；钉钉托管字段以最新远端值为准。

## 当前边界

MVP 不包含多轮对话式规划、Agent/Skill/小队编排、团队协作、云托管、移动端和完整的 Obsidian 验收工作台。优先保证个人用户可以无终端地收集、看见、安排和人工推进自己的待办。
